### План тестирования мобильного приложения  
#### 1. Введение  
##### Цели и задачи тестирования: 
**Цель:** Убедиться, что мобильное приложение «В Хосписе» работает корректно, соответствует требованиям пользователей и бизнес-целям, не содержит критических ошибок, обеспечивает качественный пользовательский опыт.   
**Задачи:**   
•	проверка функциональности приложения  
•	тестирование пользовательского интерфейса  
•	кросс-платформенное тестирование  
•	тестирование при различных сетевых условиях  
•	тестирование на удобство использования  
##### Область применения: 
**Границы приложения:**   
1.	Авторизация пользователя:  
•	Ввод логина  
•	Ввод пароля  
•	Вход в приложение  
3.	Верхняя навигационная панель (AppBar):  
•	Переход по меню приложения (главная, новости, о приложении)  
• Заголовок
•	Переход в раздел цитат (иконка бабочка)  
•	Выход из приложения  
5.	Главная страница  
•	Выпадающий список с последними активными новостями и с возможностью раскрыть более подробную информацию (нажатие на шеврон)  
• Кнопка «Все новости» с переходом в раздел «Новости»  
6.	Раздел «Новости»  
•	Меню с сортировкой, фильтрацией и кнопкой для администрирования новостей (иконка карандаш)  
•	Отображение последних новостей с возможностью раскрыть более подробную информацию (нажатие на шеврон)  
7.	Страница администрирования новостей  
•	Панель управления:   
 - сортировка по датам  
 - фильтрация новостей (по категориям, датам, статусам «Активна» и «Не активна»)  
 - создание новости (иконка +): выбор категории, ввод заголовка, даты и времени публикации, описания и выбором статуса «Активна». Возможностью сохранить или отменить создание новости  
•	Отображение новостей с возможностью удаления и внесения изменений в карточку новости  
7.	Раздел цитат (иконка бабочка)  
Отображение цитат с возможностью более подробном отображении информации (нажатие на шеврон)  
8.	Раздел «О приложении»  
•	Отображение версии приложения  
•	Ссылка на «Политика конфиденциальности»  
•	Ссылка на «Пользовательское соглашение»  
•	Отображение информации о разработчике  
#### 2. Описание приложения    
**Общее описание:**  
Приложение даёт функционал по работе с новостями хосписа и включает в себя:  
•	информацию о новостях и функционал для работы с ними;  
•	тематические цитаты;  
•	информацию о приложении.  
**Платформы:**  
Операционная система – Android  
Устройство для ручного тестирования – Poco M3 Pro (Android 13)  
Устройство для автоматизированного тестирования - эмулятор Android Studio (Medium Phone API 36.1 (Android 16.0 (“Baklava” | x86_64)))  
#### 3. Методология тестирования  
**Типы тестирования:**   
1.	Инсталляционное тестирование  
1.1.	Установка приложения.  
1.2.	Запуск приложения.  
1.3.	Удаление приложения.  
1.4.	Обновление приложения (если будет выпущена новая версия приложения)  
2.	Исследовательское тестирование  
2.1.	Определение функционала приложения  
2.2.	Изучение продукта, его поведения и потенциальные риски.  
3.	Функциональное тестирование (проверка корректности работы функциональности приложения и соответствия его заявленным требованиям)  
4.	Тестирование безопасности.  
4.1.	Тестирование авторизации.  
5.	Тестирование UI/UX  
5.1.	Проверка визуальных и интерактивных элементов интерфейса  
5.2.	Общее впечатление от взаимодействия с продуктом  
6.	Кросс-платформенное тестирование (на реальном устройстве Android 13 и эмуляторе Android 16)   
7.	Автоматизированное тестирование.  
7.1.	Автоматизация тестирования авторизации.  
7.2.	Автоматизация UI тестов.  
7.3.	Автоматизация функциональных тестов.  
**Стратегия тестирования:**  ручное и автоматизированное   
Ручное тестирование будет производиться разделов тест-кейсов: управление приложением, адаптация к изменениям в сетевых условиях, удобства использования.  
#### 4. Ресурсы  
**Команда тестирования:** QA-инженер   
**Средства тестирования:**  
Android Studio. - Среда разработки для Android проектов  
Java 17 – язык написания автотестов  
Gradle:7.0.4 – система автоматической сборки программного обеспечения  
JUnit 4.13.2 -  фреймворк для написания и запуска автоматизированных тестов на Java   
Espresso - фреймворк для автоматизации UI-тестирования Android-приложений  
Allure - инструмент для визуализации результатов тестирования  
Git - система контроля версий

#### 6. Критерии выхода  
**Критерии окончания тестирования:**  
•	Все запланированные тест-кейсы пройдены  
•	Все найденные дефекты рассмотрены  
•	Проверена работоспособность ключевых модулей (авторизация, создания, редактирования, фильтрации, удаления и отображения новостей, отображение цитат)
